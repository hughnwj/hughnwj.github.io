Chat Completion API详细参数

参数说明表

参数 是否必选 说明 默认值/取值范围
model 必选 大模型的名称 —
messages 必选 提示词（可包含角色设定） —
max_tokens 可选 返回结果的 token 数量 —
temperature 可选 采样温度。数值越小生成文本越保守；较高时文本多样 默认 1，范围 0-2
top_p 可选 控制输出文本的随机性，建议调节 temperature 默认 1，范围 0-1
n 可选 表示同一提示返回几个 Completion 默认 1
stream 可选 响应方式。False 一次性返回；True 流式返回 默认 False
logprobs 可选 返回每个 token 的前 N 个概率及对数概率 默认 null
stop 可选 指定生成文本的停止信号（一个或多个字符串） 默认 null
presence_penalty 可选 调整生成新内容的倾向性，较高值更易生成新内容 默认 0，范围 [-2, 2]
frequency_penalty 可选 调整模型重复自身的倾向性，较高值避免重复 默认 0，范围 [-2, 2]
logit_bias 可选 调整特定 token 的对数概率偏置 —
tools 可选 可调用的函数列表 —
tool_choice 可选 调用函数的策略 —
functions 废弃 可调用的函数（已废弃） —
function_call 废弃 调用函数的策略（已废弃） —

说明：Prompt 模式对比表

Prompt模式 描述 优点 缺点 适用场景
Zero-Shot 直接给出任务，不提供任何示例或上下文。 快速、简单，不需要事先准备数据；适合任务明确且模型能力较强时使用。 性能可能较低，尤其是对于复杂任务；需要模型在少量信息下能推理出答案。 基本任务，尤其是当任务说明非常清晰时；要求模型能根据任务直接给出答案。
Few-Shot 给出任务和少量的示例（通常1-5个例子），让模型学习如何解决任务。 提供上下文和实例，帮助模型更好地理解任务；提高任务复杂度时更有效。 需要选择合适的示例，示例选择不当可能影响性能；对任务的多样性要求较高。 任务明确但具有复杂性，或需要模型在少量示例的帮助下推理。
COT (Chain of Thought) 引导模型通过一步步的推理过程来得出答案，通常会增加一系列中间推理步骤。 提高模型的推理能力，尤其是复杂问题；能够更好地处理需要推理和逻辑的任务。 生成的推理链可能过于冗长或不清晰；有时可能难以判断推理链的有效性。 需要复杂推理或逻辑步骤的任务，如数学题解、复杂推理问题等。
SC (Self-Consistent) 通过多次生成不同答案并通过投票或比较来确定最一致的答案。 提高答案的准确性，减少模型单次输出的误差。 需要更多的计算资源和时间；可能导致重复的输出，增加冗余。 对准确性要求较高的任务，尤其是在高不确定性或模糊情况下。
TOT (Task-Oriented Thought) 在任务过程中，针对特定目标进行思维引导，通常会持续关注任务目标，而不是所有推理步骤。 提高效率，模型更加关注任务的最终目标；适用于长时间的任务。 可能忽视其他重要信息，只关注目标；推理过程可能不全面。 长期任务，或需要明确目标导向的任务，例如文本生成、翻译等。
Step-Back 在解答问题的过程中，如果当前步骤失败，回退到前一步重新评估或推理。 增强模型处理复杂任务时的容错能力；帮助模型在推理过程中纠错。 回退可能导致更多的计算，增加处理时间；并非所有任务都适合反向推理。 复杂任务或多步骤问题，特别是在模型可能犯错时。

Function Calling

Function Calling详细流程剖析

详解：function_response = function_to_call(**function_args)

  1. function_to_call 代表某个函数（动态决定调用哪个函数）。
  2. function_args 是一个字典，** 解包它，把键值对作为参数传入函数。
  3. function_response 接收函数返回值。

ChatGPT Plugin

ChatGPT Plugin 的核心概念

ChatGPT Plugin 的本质是一个 API 服务，让 ChatGPT 能调用你的接口，扩展功能。例如查询数据库、获取实时数据、操作第三方服务等等。
核心组成部分包括：

  • API：提供功能的接口，通常是 RESTful API
  • OpenAPI 规范：描述 API 的功能、参数、返回结果
  • Manifest 文件：定义插件的元数据（比如名称、描述、授权等）

创建插件的 Manifest 文件

创建 ai-plugin.json 文件，提供插件的元数据：

  • name_for_model: 确保简短且便于 ChatGPT 识别
  • auth: 支持 none（公开）、oauth（需要登录授权）
  • api: 指定 OpenAPI 文件地址
  • logo_url: 让插件界面更专业

部署插件服务

确保你的服务可以被外网访问，可以选择：

  • Cloudflare Pages / Workers（轻量级）
  • Vercel / Render（适合前后端一起托管）
  • AWS / GCP / Azure（大型项目）

确保 API 支持 HTTPS，否则 ChatGPT 无法调用。
